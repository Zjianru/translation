# MacOS版本BitCoinCore客户端全量数据迁移目录

作者耗时50多个小时才搞定的问题, 分享给大家。

MACOSX 系统, 迁移Bitcoin Core 数据目录的时候, 参考网上的方法试过了都不认, 无奈想到了一个损招。

[TOC]

## 1. 背景

最近在用  bitcoinj 来做验证节点, 以进行某些数据分析和调查。

但是数据存储有些问题, 想要多次迭代查询和分析全量数据, 觉得最好的方式还是使用RPC和 BitCoin Core客户端程序进行通讯。

当前日期为2025年10月16日, Bitcoin Core保存区块数据的文件总共是 `747GB` 左右.


## 2. 程序下载

下载页面为:

> https://bitcoin.org/zh_CN/download

下载后执行安装.


## 3. 数据下载

启动程序后, 进行简单的设置即可。

建议是使用自定义目录存储，如果存储空间足够的话, 可以不用限制存储区域的大小。



## 4. 踩坑指南


我是使用MACOS内置存储下载的, 前期珍惜存储空间, 限制为200GB。

这个程序会一边下载区块, 一边验证。 如果有存储空间限制, 还会对老的区块数据进行清理。

大约下载了半天, 下载到70%左右的区块, 存储部分使用了150GB的样子。

想想觉得空间小了, 所以把限制改为400GB。

又下载了半天, 下载验证完成后, 占用了350GB左右的存储。

想想觉得区块不全, 不好进行开发和测试, 所以把空间限制取消了。 

但是取消空间限制，并不会自动补全之前的数据。

无奈只好清理掉数据, 重头开始下载。

峰值时期有30MB左右的网速, 又下载了1天才下载完成。


## 5. 简单的存储性能


我这里先是使用SSD磁盘目录进行下载, 然后把数据目录复制到HDD磁盘上。


拷贝文件, 
HDD耗时为: 

```
# 大约3小时30分钟
2025年10月16日 星期四 11时29分14秒 CST
2025年10月16日 星期四 14时58分08秒 CST

# 算下来, 每分钟4GB, 每秒60MB左右。
```

磁盘分区一点要尽量多留一些冗余。 我这里复制 750GB数据, 磁盘分区总量为1000GB, 结果在复制到 80% 左右时, 拷贝速度下降了一半左右。 前期有60-80MBps, 后期掉到 30MBps 的样子.


三星T7的PSSD耗时为:

```
# 大约26分钟
2025年10月22日 星期三 12时13分35秒 CST
2025年10月22日 星期三 12时38分52秒 CST

# 折算下来, 每分钟30GB, 每秒500MB左右。
```

在大量拷贝文件的过程中, SSD的发热量比较大, 结果就降速了。 

后来我用一个移动小风扇对着磁盘吹, 感觉温度下降, 拷贝速度又上去了。


## 6. 执行步骤

1、使用SSD目录, 比如 `/Users/renfufei/BitCoreData`, BitCoinCore自动下载区块数据, 大约耗时24小时。 我这里因为踩坑, 耗时翻倍, 达到了48小时。 如果网速不好, 耗时多天也是可能的。
2、关闭 Bitcoin Core 程序, 避免后续复制的文件不一致或者损坏。
3、复制数据目录到另一个磁盘的目录, 比如 `/Volumes/BitCoreDisk/BitCoreData` , 因为是HDD, 这个过程我这里耗时大约3个小时多, 合计200分钟左右。
4、确认文件无误之后, 把最初的data保存目录改个名字, 让后续启动BitCoinCore时提示修改目录。 比如改名为 `BitCoreData_BAK`, 直接改名字而不是删除目录的好处是, 避免后续哪里操作出问题导致数据不在了又得重新下载。 当然, 备份后再删除原来目录也是可以的。
5、打开 Bitcoin Core 程序, 此时会提示选择目录, 选择新目录旁边的一个其他目录(这个目录可以不存在), 比如 `/Volumes/BitCoreDisk/BitCoreFullData`, 稍等一会儿, 程序对应的数据目录已经修改到新磁盘内了。
6、退出 Bitcoin Core 程序。
7、调换新磁盘中的目录名字， 比如把 `BitCoreFullData` 换成 `BitCoreFullData_BAK`。 再把 `BitCoreData` 目录改名成 `BitCoreFullData`。 同一个磁盘内的目录改名, 不会涉及文件的大量复制。
8、重新启动 Bitcoin Core 程序, 启动之后会进行验证，一般几分钟或者几十分钟就好了。
